次の 3 点が明確に守られているかレビューし、守られていなければ修正すること

1. **これが最も大事** `.claude/rules/testing/test-case-example.md`の例のように生きたドキュメントになっていること
2. 必ず **" should 【動作】 when 【条件】"** を日本語で書いたフォーマットで記載し、ブラックボックスなテストケースになっていること。実装の詳細を検証する内容になっていないこと
3. 特に重要な機能が重点的にテストケースになっており、取るに足らないことがテストケースになっていないこと
4. 過剰な量のテストケースになっていないこと。類似したテストケースは圧縮して削減すること。少なければいいというわけでもなく、多ければいいというわけでもない。質のいいテストケースが必要なだけあることが大事。
