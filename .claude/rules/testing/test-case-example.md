## いいテストケースの具体例

以下のように、describe を使い抽象度ごとにテストケースを書き、生きたドキュメントにすること
3 年後アサインされた人が見ても、何を目的にテストしているかがわかる抽象度の高い describe をかき、その具体の動きを test として記述すること

describe("FizzBuzz", () => {
  // 仕様を抽象度の高い状態の仕様で分類する
  // 3年後アサインされた人が、テストをドキュメントとして参照でき、配下のテストケースが何をテストしているかがわかるようにする

    // 抽象度の低い状態の具体的な具体的な仕様のケースを入れる
    test("3を渡したら”Fizz”を返す", () => {
      // arrange
      // act
      // assert
    });
    // 抽象度の低い状態の具体的な具体的な仕様のケースを入れる
    test("6を渡したら”Fizz”を返す", () => {
      // arrange
      // act
      // assert
    });
  });

  // 仕様を抽象度の高い状態の仕様で分類する
  // 3年後アサインされた人が、テストをドキュメントとして参照でき、配下のテストケースが何をテストしているかがわかるようにする
  describe("5の倍数を渡したら”Buzz”を返す", () => {
    // 抽象度の低い状態の具体的な具体的な仕様のケースを入れる
    test("5を渡したら”Buzz”を返す", () => {
      // arrange
      // act
      // assert
    });

    // 3の倍数で実装の正しさがある程度確保されているので、無駄なケース(10, 20)は入れない
  });

  // 仕様を抽象度の高い状態の仕様で分類する
  // 3年後アサインされた人が、テストをドキュメントとして参照でき、配下のテストケースが何をテストしているかがわかるようにする

    // 抽象度の低い状態の具体的な具体的な仕様のケースを入れる
    test("15を渡したら”FizzBuzz”を返す", () => {
      // arrange
      // act
      // assert
    });
  });

  // 仕様を抽象度の高い状態の仕様で分類する
  // 3年後アサインされた人が、テストをドキュメントとして参照でき、配下のテストケースが何をテストしているかがわかるようにする
  describe("それ以外の数を渡したらそのまま文字列に変換する", () => {
    // 抽象度の低い状態の具体的な具体的な仕様のケースを入れる
    test("1を渡したら”1”を返す", () => {
      // arrange
      // act
      // assert
    });
    test("2を渡したら”2”を返す", () => {
      // arrange
      // act
      // assert
    });
  });
});
